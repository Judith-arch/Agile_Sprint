# Deployment

## Production architecture

```
Frontend (Vercel)  <-->  Backend (Railway or Render)
```

The frontend and backend are deployed separately. The frontend on Vercel (static), the backend on a Node.js-compatible service such as Railway or Render.

---

## Backend — Railway or Render

### Option A: Railway

1. Create an account at [railway.app](https://railway.app)
2. New project from GitHub — select the `Agile_Sprint` repository
3. Set the root directory: `Modulo 5/server`
4. Add environment variable: `PORT=3000`
5. Railway automatically detects Node.js and runs the `start` script from `package.json`
6. Copy the public URL generated by Railway (e.g. `https://agile-sprint.up.railway.app`)

### Option B: Render

1. Create an account at [render.com](https://render.com)
2. New Web Service from GitHub
3. Root directory: `Modulo 5/server`
4. Build command: `npm install`
5. Start command: `npm start`
6. Add environment variable: `PORT=3000`
7. Copy the public URL

---

## Frontend — Vercel

1. Create an account at [vercel.com](https://vercel.com)
2. Import the repository `Judith-arch/Agile_Sprint`
3. Configure:
   - **Root directory:** `Modulo 5`
   - **Framework preset:** Vite
   - **Build command:** `npm run build`
   - **Output directory:** `dist`
4. Add environment variable:
   ```
   VITE_API_URL=https://YOUR-BACKEND.up.railway.app/api/v1/tasks
   ```
5. Deploy

---

## Environment variables

| Variable | Environment | Value |
|---|---|---|
| `VITE_API_URL` | Frontend (Vercel) | Public backend URL + `/api/v1/tasks` |
| `PORT` | Backend (Railway/Render) | `3000` or whatever the platform assigns |

**Important:** `VITE_API_URL` must point to the deployed backend, not to `localhost`. If not configured, the frontend will not be able to communicate with the API in production.

---

## Considerations

- The backend uses an **in-memory store** — data is lost every time the server restarts. In production this means tasks do not persist between deploys or restarts.
- For a real production setup, a database should be added (MongoDB, PostgreSQL, etc.)
- Vercel serves the frontend as static files — React Router routing works correctly because Vite generates an `index.html` that Vercel configures as a fallback.
