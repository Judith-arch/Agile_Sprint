# TaskFlow

> Fullstack task management with Kanban view, drag & drop and real-time statistics.

A web application for personal and team task management. Create, organize and track tasks through an interactive Kanban board, with support for priorities, deadlines, assignment, dark mode and progress statistics.

## Features

1. Kanban board with drag & drop between columns (To-do, Doing, Review, Done, Cancelled)
2. Statistics panel with circular progress and distribution bars by status
3. Full dark mode with localStorage persistence
4. Search filters and date-based sorting
5. Detailed statistics page with priority breakdown and critical tasks

## Tech stack

### Frontend

| Technology | Usage |
|---|---|
| React 18 + TypeScript | Typed components and state |
| Tailwind CSS v4 | Styles and dark mode |
| React Router v6 | SPA navigation |
| @dnd-kit | Drag & drop between Kanban columns |
| date-fns | Date calculation and formatting |
| Vite | Bundler and dev server |

### Backend

| Technology | Usage |
|---|---|
| Node.js + Express | HTTP server and REST API |
| Layered architecture | routes → controllers → services |
| In-memory store | Task data persistence |
| CORS | Cross-origin communication with frontend |

## Getting started

```bash
git clone https://github.com/Judith-arch/Agile_Sprint.git
cd Agile_Sprint
```

### Backend

```bash
cd "Modulo 5/server"
npm install
npm run dev
# Server running at http://localhost:3000
```

### Frontend

```bash
cd "Modulo 5"
npm install
npm run dev
# App running at http://localhost:5173
```

> Make sure the `.env` file in the root contains:
> ```
> VITE_API_URL=http://localhost:3000/api/v1/tasks
> ```

## Deploying to production

### Backend (Railway)

1. Create a new project on [railway.app](https://railway.app) from GitHub
2. Root directory: `Modulo 5/server`
3. Start command: `npm start`
4. Add environment variable: `PORT=3000`
5. Copy the public URL generated by Railway

### Frontend (Vercel)

1. Import the repo `Judith-arch/Agile_Sprint` on [vercel.com](https://vercel.com)
2. Root directory: `Modulo 5`
3. Framework preset: Vite
4. Add environment variable: `VITE_API_URL=https://YOUR-BACKEND-URL/api/v1/tasks`
5. Deploy
